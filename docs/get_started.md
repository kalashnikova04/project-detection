## Запуск проекта
В терминале:

1. Клонируйте репозиторий
```git clone -b master https://github.com/kalashnikova04/project-detection.git project-detection-v-dc```

2. ```cd project-detection-v-dc```
3. Создайте файл ```.env``` с переменными из файла ```.env.example```. Замените значения на соответствующие логины/пароли.
4. ```docker-compose build```
5. ```docker-compose up```

В браузере перейдите по адресу http://localhost:8099/

## Функционал сервиса

На всех страницах сервиса в заголовке есть панель навигации. Всего 4 раздела:
- Upload - перенаправляет в корневой каталог
- Train - `href="/train/"`
- See Tasks - `href="/task_status/"`
- Inference - `href="/inference/"`

Рассмотрим каждый раздел отдельно:

http://localhost:8099/ - форма для загрузки изображений. После нажатия кнопки ```Upload``` загруженные изображения появятся на экране. Чтобы перейти к следующему этапу нажмите на кнопку ```Get preds```, которая перенаправит запрос в раздел ```model_mode/```

http://localhost:8099/model_mode/ - промежуточная страница, которая информирует переведена ли модель в режим обучения или инференса. Нажав на ```status```, вы перейдете в раздел ```task_status/```

http://localhost:8099/task_status/ - страница для просмотра статуса задач из очереди, в нашем случае задач обучения и инференса.

http://localhost:8099/inference/ - страница для просмотра детектированных объектов на загруженных изображениях.

http://localhost:8099/train/ - тренировка модели Yolov8.

http://localhost:8099/admin/ - админ-панель для суперпользователя, данные которого заданы в `.env` файле.

http://localhost:15672/ - UI сервиса RabbitMQ, данные для входа заданы в `.env` файле.
